# Product Requirements & User Flow Specifications

This directory contains product requirement documents (PRDs), catalog management procedures, user flow maps, and founder feedback constraints.

## Key Files

* **[USER-FLOW.md](USER-FLOW.md)**: **Core Platform User Loop** documenting browse, search, location selection, item details, and contribution actions.
* **[SEO.md](SEO.md)**: Search Engine Optimization meta guidelines and dynamic metadata structures.
* **[CATALOG-STEWARDSHIP-WORKFLOW.md](CATALOG-STEWARDSHIP-WORKFLOW.md)**: Workflow specification for catalog moderators managing places, items, aliases, and observation data.
* **[COMMUNITY-TRUST-AND-CONTRIBUTION-LOOP.md](COMMUNITY-TRUST-AND-CONTRIBUTION-LOOP.md)**: Specifications for crowdsourced reporting and verification loops.
* **[GENERAL-SEARCH-AND-DECISION-ENGINE.md](GENERAL-SEARCH-AND-DECISION-ENGINE.md)**: Strategic vision for AI-routed natural-language search grounded only in typed WetinDey evidence.
* **[FOUNDER-FEEDBACK.md](FOUNDER-FEEDBACK.md)**: Founder review feedback logs and UX corrective actions.
