# Founder Feedback Register

This register preserves every product complaint and request raised in the Founder
orchestrator thread and reconciles each one to current repository evidence. `LANES.md`
remains authoritative for execution ownership. A commit proves source history, not live
behavior; a push proves remote history, not a Ready deployment; and a Ready deployment
does not prove the requested experience without relevant runtime evidence.

## Lifecycle fields

- **Planning / decision** records whether direction is documented, accepted, deferred, or
  still awaiting an ADR/decision.
- **Implementation** records source state independently of planning.
- **Live evidence** records runtime, browser, deployment, or provider evidence separately.
- **Owner / lane** names the current accountable lane; `pathless` grants no edit authority.
- **Evidence** names concrete ADRs, commits, pushes, deployments, or refuter verdicts.
- **Next gate** is the first unresolved condition and must not be inferred as complete.

## Mechanical reconciliation rule

Program Management updates this file whenever any of these events occurs:

1. `LANES.md` changes a complaint’s owner, exact paths, dependency, or lifecycle status.
2. A scoped implementation or governance commit lands locally or on remote main.
3. A deployment for the relevant candidate reaches `Ready` or `Error`.
4. A static, runtime, provider, visual, accessibility, security, or documentation refuter
   changes verdict.

Each update must change only the affected lifecycle fields, cite concrete evidence, preserve
the Founder’s complaint wording, and keep commit/push/deployment/runtime claims separate.
A request is never “live” from an ADR, source commit, static PASS, push, or deployment
alone; relevant live evidence must be recorded explicitly.
